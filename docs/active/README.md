# Active Work

Current work in progress.

## Purpose

Contains:
- Active implementation plans
- Current focus areas
- Work-in-progress documentation

## Structure

```
docs/active/
├── README.md          ← This file
├── current-task.md    ← What's being worked on now
└── next-up.md         ← What's planned next
```

## Workflow

1. Start work → Create plan in `docs/active/`
2. Complete work → Move to `docs/completed/YYYY-MM/`
3. Update `README.md` with current status

## Notes

- This folder should be **empty or minimal** when no active work
- Clean up after completing tasks
- Not for storage - use `docs/reference/` or `docs/sessions/`
