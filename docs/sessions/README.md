# Session Notes

Historical records of development sessions.

## Purpose

Session notes capture:
- Work in progress
- Discoveries made during development
- Decisions and rationale
- Debugging notes

## Naming Convention

`YYYY-MM-DD-topic.md` - Date-prefixed for chronological ordering

## Notes

- **Read-only after creation** - Session notes are historical records
- Not for active planning - use `docs/active/` for current work
- Reference material for future debugging

## Example

```markdown
# Session: 2026-03-28 - Right Panel Visual Fixes

## Context
Fixing visual issues in right panel...

## Discoveries
- Found border issues in ContextSection...
- Textual's compose_add_child pattern...

## Decisions
- Use widget-level CSS instead of app-level...
```
